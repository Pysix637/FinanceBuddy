# Пользователи и бизнес‑процессы

## Роли
**Пользователь** — владелец финансового дневника.

## Основные бизнес‑процессы

### Добавление операции
1. Пользователь выбирает тип операции.
2. Указывает категорию, дату, описание и сумму.
3. Я проверяю корректность ввода.
4. Система сохраняет операцию и пересчитывает сводку.

### Просмотр и анализ операций
1. Пользователь открывает список операций.
2. Система отображает операции.
3. Система показывает итоговые суммы.

### Удаление операции
1. Пользователь выбирает операцию в таблице.
2. Нажимает кнопку удаления.
3. Система удаляет операцию и пересчитывает сводку.

## Нефункциональные требования
- быстрый отклик интерфейса;
- понятный и простой экран на первом этапе;
- удобное расширение проекта за счёт MVVM и сервисов.
